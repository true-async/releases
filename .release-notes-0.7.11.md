# TrueAsync PHP 0.7.11 (php 8.6)

## Highlights

- **`pcntl_fork()` is now safe under an active reactor.** A forked child inherited the
  parent's libuv event loop (shared epoll backend), so its blocking I/O spun on stale
  events — `php artisan tinker` and any PsySH REPL under laravel-spawn started but never
  evaluated input and exited with no error. The child now reinitializes its reactor via
  `uv_loop_fork()`, and `pcntl_fork()` refuses to fork while coroutines other than the
  main one are running.

## Components (php-src trueasync-0.7.8, php-async v0.7.10)

**ABI change: TrueAsync ABI v0.23.0 -> v0.24.0** — two additive fork hooks
(`zend_async_before_fork_fn`, `zend_async_after_fork_child_fn`).

### Fixed
- **php-src/php-async: `pcntl_fork()` under an active reactor broke the child.** The child
  inherited the parent's libuv loop (shared epoll); blocking I/O in the child spun on
  stale events and REPLs never processed input. The child now reinitializes the loop via
  `uv_loop_fork()`, giving it an independent reactor. Fixes `php artisan tinker` / PsySH
  exiting silently under laravel-spawn.

### Changed
- **`pcntl_fork()` now throws when a coroutine other than the main one is running.** Its
  in-flight reactor state and worker threads cannot survive `fork()`, so forking is
  refused with a clear error instead of corrupting the child.
