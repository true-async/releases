# TrueAsync PHP 0.7.3 (php 8.6)

## Async extension

### Fixed

- **#164 `Async\runtime_stats()` aborted the process when called before the scheduler starts.** A call at the top level — before the first `spawn` — read uninitialised circular buffer headers. `circular_buffer_count()` asserted `head < capacity` (0 < 0) and aborted on debug builds; on release builds the counts were garbage. Fixed by returning 0 from `circular_buffer_count()` for an uninitialised buffer (capacity 0), clamping the four queue-count reads in `runtime_stats()` the same way the pool-capacity read already was, and short-circuiting the whole function to a zeroed snapshot when `ZEND_ASYNC_SCHEDULER` is NULL. Regression tests added for the top-level (pre-scheduler) and in-coroutine cases.

- **#165 Cross-thread transfer of objects with a materialised property table was incorrectly rejected.** `thread_transfer_object()` counted every slot in `zend_object.properties` as a dynamic property, including `IS_INDIRECT` slots that back declared properties. An object whose property table had been materialised by `var_dump`, `get_object_vars`, `foreach`, or an array cast was therefore rejected even if it had no actual dynamic properties. Fixed by iterating the property table and counting only non-indirect (genuine dynamic) entries. Regression tests added for every materialisation path (sync), a real cross-thread round-trip, and a `$this`-bound closure whose object had a materialised table.

## Installation

### Windows (PowerShell)

```powershell
irm https://raw.githubusercontent.com/true-async/releases/master/installer/install.ps1 | iex
```

To also install FrankenPHP non-interactively:

```powershell
$env:INSTALL_FRANKENPHP="true"; irm https://raw.githubusercontent.com/true-async/releases/master/installer/install.ps1 | iex
```

### Docker

```bash
docker pull trueasync/php-true-async:0.7.3-php8.6
docker pull trueasync/php-true-async:latest
```
