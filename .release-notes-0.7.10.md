# TrueAsync PHP 0.7.10 (php 8.6)

## Highlights

- **Clean pool shutdown no longer leaks or crashes.** `Async\graceful_shutdown()` of
  an `HttpServer` worker pool left each worker's cross-thread completion trigger armed
  on the parent reactor (loop-alive assert on debug / leaked libuv handle on release),
  and could hard-crash on macOS ARM64 by double-disposing the parent's wait event.
  Both are fixed; the underlying cross-thread future teardown is now race-safe.

## Components (php-async v0.7.9, server v0.9.3)

No ABI change — still TrueAsync ABI v0.23.0.

### Fixed
- **php-async: race-safe `remote_future_dispose`.** Disposing a remote future while its
  producer thread was still completing it raced — the cross-thread trigger was closed
  without first marking the shared state completed or detaching `target_future`, so a
  concurrent completion could ring a closing handle or resolve a freed future. Teardown
  now happens under the shared-state mutex, so a late completion is a clean no-op.
- **server: pool parent reaps its per-worker completion futures on shutdown
  (true-async/server#93).** They were never disposed, so their cross-thread wakeup
  triggers lingered armed on the parent reactor after `graceful_shutdown()` — a
  loop-alive assert (debug) / leaked handle (release). Disposal is race-safe via the
  php-async fix above.
- **server: use-after-free from double-disposing the pool parent's `all_done` wait
  event.** The waker already disposes it; disposing again in cleanup was a UAF —
  a hard `EXC_BAD_ACCESS` on macOS ARM64 release.

## Installation

### Windows (PowerShell)

```powershell
irm https://raw.githubusercontent.com/true-async/releases/master/installer/install.ps1 | iex
```

To also install FrankenPHP non-interactively:

```powershell
$env:INSTALL_FRANKENPHP="true"; irm https://raw.githubusercontent.com/true-async/releases/master/installer/install.ps1 | iex
```

### Docker

```bash
docker pull trueasync/php-true-async:0.7.10-php8.6
docker pull trueasync/php-true-async:latest
```
