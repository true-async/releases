# TrueAsync PHP 0.7.5 (php 8.6)

## Highlights

- **Android platform support.** `libphp.a` + headers now cross-compile for Android (arm64-v8a, x86_64) via `scripts/build-android.sh`, for embedding TrueAsync PHP in Android apps through JNI. This required a new core API — `ZEND_ASYNC_CALL_ON_MAIN_STACK` (ABI v0.21) — that runs foreign calls (JNI into ART, FFI) on the main coroutine's OS-thread stack instead of a fiber stack, where such runtimes' stack-pointer checks would otherwise misfire (e.g. ART's `StackOverflowError` at the JNI→Java boundary), plus Bionic build fixes (TSRM TLS model, `getdtablesize`) and Android-specific configure defaults (opcache and Zend signals disabled — they conflict with ART's JIT thread pool / SIGSEGV null-check handler).

## Async extension

### Added

- **`zend_async_call_on_main_stack_fn` (ABI v0.21).** Extension-side implementation of the new core API: a naked SP-only stack switch (arm64 + x86_64, the `asmcgocall` pattern) to run a callback on the main coroutine's stack; passthrough if already on main.

### Fixed

- **(true-async/server#2) Reactor: io-completion exception freed mid-broadcast (UAF/heap corruption).** `io_pipe_read_cb`/`io_pipe_write_cb` broadcast `req->base.exception` to every callback subscribed on the shared `io->event`, but the exception is owned by `req` and dies with it. A consumer — e.g. a connection's persistent multishot read listener — could legitimately release its req's exception and dispose the req inside the same notify pass, while a sibling write-filter for a parked writer still forwarded that same object to `async_coroutine_resume`, add-reffing freed memory. Fixed in `libuv_reactor.c` by having the producer capture the exception into a local, `GC_ADDREF` before `NOTIFY` and `OBJ_RELEASE` after.

## php-src

### Added

- **`ZEND_ASYNC_CALL_ON_MAIN_STACK` API.** See Highlights above.

### Fixed

- **Android (Bionic) build fixes.** `getdtablesize()` shim via `sysconf(_SC_OPEN_MAX)`; TSRM takes the safe default TLS model and skips the gottpoff/ntpoff inline asm so `dlopen()` works under Bionic.

## Installation

### Windows (PowerShell)

```powershell
irm https://raw.githubusercontent.com/true-async/releases/master/installer/install.ps1 | iex
```

To also install FrankenPHP non-interactively:

```powershell
$env:INSTALL_FRANKENPHP="true"; irm https://raw.githubusercontent.com/true-async/releases/master/installer/install.ps1 | iex
```

### Docker

```bash
docker pull trueasync/php-true-async:0.7.5-php8.6
docker pull trueasync/php-true-async:latest
```

### Android

```bash
./scripts/build-android.sh --abi arm64-v8a
```

Produces `libphp.a` + headers for embedding in an Android APK via JNI. See `build-config.json`'s `android` section for NDK version, API level and supported ABIs.
