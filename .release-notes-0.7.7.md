# TrueAsync PHP 0.7.7 (php 8.6)

## Highlights

- **Live hot reload of the worker pool** — `HttpServer::reload()` rotates the worker
  pool without dropping the listen sockets, so changed code is picked up with zero
  downtime. Ships with built-in dev (filesystem watch) and prod (`SIGHUP`) triggers.
- **Reactor-aware signal handling** — `pcntl` / Zend signal handlers now run on the
  main coroutine instead of an arbitrary interrupted context (new `zend_async_sigaction`
  hook, **TrueAsync ABI v0.23.0**).
- **Worker thread-stack leak fixed** — every exited pool worker used to leak its 8 MB
  stack; worker threads are now detached portably.

## Server extension (v0.9.2)

### Added
- **`HttpServer::reload()`** — rotate the worker pool live, without dropping the listen
  sockets. An epoch beacon fans out to worker clones; each drains its in-flight requests,
  stops, and its replacement re-runs the bootloader. Reload is serialized and its
  lifecycle is logged (`reload.start` / `reload.done`).
- **Built-in reload triggers**
  - `HttpServerConfig::enableHotReload(array $watchPaths, array $extensions = ['php'], int $debounceMs = 300, int $maxHoldMs = 2000)` — dev: recursive `Async\FileSystemWatcher` per path; a debounced change invalidates opcache and calls `reload()`.
  - `HttpServerConfig::enableReloadOnSignal(bool $enabled = true)` — prod: a persistent `SIGHUP` handler that calls `reload()`.
- **Reload under the reactor pool** — works with `TRUE_ASYNC_SERVER_REACTOR_POOL=1` via a
  worker-inbox retirement protocol; connections re-home on their next request (#100).

### Fixed
- ~10 KB leaked per reload rotation (worker transit shell C-state + side-cars).
- Heap corruption on rotation under the gated reactor pool (global worker registry was
  torn down from a worker thread; teardown is now parent-only).
- macOS build: TSRM mutex instead of `uv_mutex` in the worker registry.

## Async extension / engine (php-async v0.7.6, php-src trueasync-0.7.6)

### Added
- **`ThreadPool` rolling reload matured into a single `reload()` ABI** — channel-swap
  reload replaces the earlier `respawnWorker()` / `requestWorkerExit()` primitives;
  hardened with identity-gated exit tokens and serialized+coalesced overlapping calls.
  In-flight tasks always complete.
- **`FileSystemWatcher` recursive watch + built-in debounce** on Linux/inotify and
  BSD/kqueue (`debounceMs` / `maxHoldMs` collapse a change burst into a single event).
- **`zend_async_sigaction` hook (ABI v0.23.0)** — `pcntl` / Zend signal handlers routed
  through the reactor, running on the main coroutine.

### Fixed
- **Every exited pool worker leaked its 8 MB stack** — worker threads are now detached
  portably (`pthread_detach` on unix, `uv_thread_detach` where libuv ≥ 1.50).

## Validation

`php 8.6.0-dev … TrueAsync ABI v0.23.0` (this stack) passed the downstream server suite —
286/287 phpt + 69/69 laravel-spawn; reactor-pool reload paths green under
`TRUE_ASYNC_SERVER_REACTOR_POOL=1`.

## Installation

### Windows (PowerShell)

```powershell
irm https://raw.githubusercontent.com/true-async/releases/master/installer/install.ps1 | iex
```

To also install FrankenPHP non-interactively:

```powershell
$env:INSTALL_FRANKENPHP="true"; irm https://raw.githubusercontent.com/true-async/releases/master/installer/install.ps1 | iex
```

### Docker

```bash
docker pull trueasync/php-true-async:0.7.7-php8.6
docker pull trueasync/php-true-async:latest
```
