# TrueAsync PHP 0.7.12 (php 8.6)

## Highlights

- **Correctness & interop across the HTTP stack.** Duplicate request headers are now
  combined per RFC 9110 §5.3 on HTTP/1.1, HTTP/2 and HTTP/3 (`Cookie` joined with `"; "`,
  others with `", "`) instead of the last value silently winning; WebSocket
  permessage-deflate honours the client's `server_max_window_bits` offer (RFC 7692) so
  window-capped peers interoperate.
- **Hardening & a gRPC streaming fix.** The inbound WebSocket message FIFO is now bounded
  (8× `max_message_size`, close 1013 on overflow) so a flooding peer can't grow the queue
  without limit, and an already-complete request is no longer upgraded to streaming — a
  deadlock that could park a gRPC handler until the connection died.

This is a server-only update (php-src, php-async and the ABI are unchanged).

## Components (php-src trueasync-0.7.8, php-async v0.7.10, server v0.10.0)

**No ABI change** — TrueAsync ABI stays at v0.24.0.

### Fixed
- **server: duplicate request headers combined per RFC 9110 §5.3** across H1/H2/H3, via a
  shared store path — previously the last duplicate value won.
- **server: WebSocket permessage-deflate `server_max_window_bits` negotiation (RFC 7692)** —
  the response now reflects the peer's window-bits offer instead of always advertising max.
- **server: inbound WebSocket message FIFO bounded** (8× `max_message_size`) — a peer that
  floods faster than the handler drains is closed with 1013 rather than growing unbounded.
- **server: an already-complete request is no longer upgraded to streaming** — fixes a
  gRPC deadlock where the handler parked on a finished request until the connection closed.
- **server: reactor-spawned listeners get their own per-listener counters slice**, fixing a
  telemetry race between sibling reactors.

### Performance
- **server: HTTP/3 reactor-mode request bodies built persistent from the first byte** and
  the wire body is adopted instead of re-copied; **gRPC `readMessage` reassembly compaction
  amortized**; assorted transport hot-path micro-cleanups.

### Changed
- **server: configure fails fast on a non-ZTS (NTS) PHP** — the threaded worker pool
  requires ZTS by design, so the build errors early instead of producing a broken extension.
